A personal finance manager for tracking income and expenses, analyzing spending behavior, and generating financial reports. Built with a focus on backend engineering, improving performance through SQL indexing and caching, ensuring data integrity with atomic database transactions, hardening security with CSP and HTTP security headers, and generating scheduled reports using Hangfire background jobs. The application follows a clean layered architecture designed for maintainability and testability.

🔑 Key Features
  • Income & Expense Tracking: Record financial transactions and classify them into categories.
  • Multi-Wallet Management: Track money across different wallets and use one for each transaction.
  • Dashboards: Visualize financial behavior via interactive charts and aggregate data points.
  • Scheduled Reporting: Automatically generate and send reports via email (weekly, monthly, or custom schedules).
🚀 Engineering Highlights
  • 🛡️ Atomic Database Transactions (ACID): Ensures full data integrity by wrapping multi-table operations into single units, preventing partial updates and inaccurate financial balances.
  • 🗄️ Database Indexing: Added a composite index on (UserId, Date) in the Transactions table to optimize frequently filtered queries.
  • 🏎️ High-Performance EF Core: Used ExecuteUpdateAsync for bulk database updates without loading entities into memory, and AsNoTracking for read-only queries to eliminate Change Tracker overhead, and optimized LINQ queries.
  • ⚡ Optimized Caching: Reduced database round-trips for frequently accessed data using IMemoryCache and invalidating the cache on updates.
  • ⛓️ Scheduled Reports: Implemented user-defined report scheduling using Hangfire. At the scheduled time, a background job generates the report and delivers it via email. Handled time-zone conversions so schedules align with the user's local time.
  • 🔐 Security: Secured the app against XSS and injection attacks using Content Security Policies (CSP) and HTTP security headers. Added global CSRF protection on all data-changing requests. Enabled signing in with Google (OAuth 2.0).
  • 🧪 Testing & Error Handling: Verified the core logic by writing Unit Tests (xUnit/Moq). Implemented a Global Exception Middleware and added logging using Serilog to log the exceptions or other info.
🏗️ System Design & Architecture

The system follows an N-Tier Layered Architecture using Repository & Service Layer Patterns. This structure ensures a clear separation of concerns, decouples business logic from data access, and ensures the codebase is testable and maintainable according to SOLID principles.

🧠 Challenges & Learnings

Through this project, I evolved from developing basic CRUD apps to engineering production-grade software, with a focus on quality attributes like data integrity, performance, security, and maintainability to meet professional standards.

  • Financial Accuracy: Used Decimal instead of Float for all money attributes to prevent rounding errors.
  • Clean Structure: Used ViewModels to keep the models private and easier to update without breaking the app.
  • Data Integrity: Automated cascading updates on deleting a wallet or changing category type.
  • Global Logic: Handled Time-Zones so that all dates and schedules are correct for users worldwide.
